# Demo recording

The README's "Demo" section will link to a short screen recording from
this directory once it's recorded. Target spec:

- **~60 seconds** total
- **MP4** (H.264) and **GIF** versions
- **Mobile-frame mockup** is fine but not required — a clean browser
  recording at desktop or device size both work
- **What to show** (in order):
  1. Login screen with the macOS window chrome (5s)
  2. Terminal connected; run `git status` to show real shell (5s)
  3. Run `claude -p "add a JSDoc comment to hello.js explaining what it does"` (10s)
  4. Watch the file panel auto-light its dirty indicator (5s)
  5. Open the panel, tap the file, see the diff (10s)
  6. Type a commit message, tap commit & push, see the steps (10s)
  7. Tap the memory button, save a checkpoint, close (5s)
  8. End frame: GitHub URL + "self-host: link" (5s)
- **No music**, soft keyboard sounds OK
- **Quiet UI**: hide notifications, dock, menubar before recording

Save here as `demo.mp4` and `demo.gif` (≤6MB for GitHub render).
Reference from the README's `## Demo` section.
